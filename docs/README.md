# Documentación de Swayp

Índice de la documentación del proyecto. Se actualiza en cada commit relevante.

- [`ARCHITECTURE.md`](./ARCHITECTURE.md) — diseño completo de la arquitectura (backend, frontend, esquema de datos, contrato de API, UX, notificaciones, decisiones de alcance).
- [`ROADMAP.md`](./ROADMAP.md) — ideas a futuro, funcionalidades pospuestas conscientemente, mejoras pendientes.
- [`TODO.md`](./TODO.md) — tareas a corto plazo, ligadas a la fase actual del roadmap de implementación.
- [`CHANGELOG.md`](./CHANGELOG.md) — registro cronológico de avances, bugs corregidos y decisiones tomadas commit a commit.
- [`decisions/`](./decisions) — registro de decisiones técnicas (ADR), una por decisión relevante, con su motivo y alternativas descartadas.

## Convención de commits
Prefijo tipo Conventional Commits (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`). Se actualiza `CHANGELOG.md` en el mismo commit cuando el cambio es relevante para el histórico del proyecto.
